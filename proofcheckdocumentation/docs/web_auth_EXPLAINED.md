# `proofcheck/web/auth.py` — Explained

> Optional, stdlib-only authentication: salted PBKDF2 password hashing + stateless HMAC-signed session cookies, plus the FastAPI `current_user` dependency. Off by default; opt in with `PROOFCHECK_AUTH=on`.

## Purpose
Adds user authentication to the web layer **without any external identity provider or extra dependency** — classic crypto from `hashlib`/`hmac`, nothing AI/ML. It is deliberately opt-in: disabled by default so the CLI flow and the disposable single-user SPA keep working with zero configuration (and the existing test-suite stays green). When enabled, it gates the protected API routes and attributes run history to the logged-in user. All auth lives here in `web/`; the core engine and `pipeline.run()` are untouched.

## Configuration (all via environment)
| Variable | Default | Meaning |
| --- | --- | --- |
| `PROOFCHECK_AUTH` | off | `on` enables auth gating. |
| `PROOFCHECK_SECRET` | autogenerated | HMAC signing key for session tokens (set it to persist sessions across restarts). |
| `PROOFCHECK_SESSION_HOURS` | `12` | Session lifetime. |
| `PROOFCHECK_ADMIN_USER` / `PROOFCHECK_ADMIN_PASSWORD` | — | Bootstrap admin, created on startup if no users exist. |
| `PROOFCHECK_ALLOW_REGISTER` | off | `on` exposes `POST /api/auth/register`. |

## Dependencies
- **Imports (external):** `base64`, `hashlib`, `hmac`, `os`, `secrets`, `datetime`; `fastapi.Cookie`, `fastapi.HTTPException`.
- **Imports (internal):** `from . import store` (user lookups + creation).
- **Used by:** `proofcheck/web/app.py` — `current_user` dependency on protected routes, `auth_enabled()` in `/api/health`, and the `/api/auth/*` routes.

## Line-by-line / block-by-block breakdown

### Constants
`SESSION_COOKIE = "proofcheck_session"`, `ANONYMOUS = "anonymous"`, `_PBKDF2_ITERATIONS = 240_000`, `_PBKDF2_ALGO = "sha256"`. `_FALLBACK_SECRET = secrets.token_hex(32)` is generated **once per process** and used only when `PROOFCHECK_SECRET` is unset, so tokens stay valid within a run but you should set the env var in production to survive restarts.

### Config helpers
`auth_enabled()`, `registration_enabled()`, `_secret()`, `_session_seconds()`, `_now()` all read the environment **fresh** so toggling env (including via `monkeypatch` in tests) takes effect per-request without reimporting the app.

### Password hashing — `hash_password` / `verify_password`
```python
digest = hashlib.pbkdf2_hmac("sha256", password.encode(), salt_bytes, 240_000)
```
`hash_password` returns `(hash_hex, salt_hex)`, generating a fresh 16-byte salt when none is supplied. `verify_password` re-hashes the candidate with the stored salt and compares with `hmac.compare_digest` (constant-time, resists timing attacks).

### Session tokens — `make_token` / `verify_token`
Stateless tokens of the form `base64(username:expiry).base64(hmac_sig)`. `make_token` signs `username:expiry` with the secret; `verify_token` recomputes the signature, `compare_digest`-checks it, then validates the expiry — returning the username only for a valid, unexpired token, else `None`. No server-side session store is needed.

### `current_user` (FastAPI dependency)
```python
def current_user(proofcheck_session: str | None = Cookie(default=None)) -> str:
    if not auth_enabled():
        return ANONYMOUS
    username = verify_token(proofcheck_session)
    if username is None:
        raise HTTPException(status_code=401, detail="Authentication required.")
    return username
```
The single gate used across protected routes. **Auth off → everyone is `anonymous`** (so nothing breaks and history still works). **Auth on → a valid session cookie is required**, else `401`.

### `authenticate` / `register_user` / `bootstrap_admin`
- `authenticate(username, password)` looks up the user and verifies the password; if the user doesn't exist it still hashes once to avoid leaking valid usernames via timing.
- `register_user(username, password)` validates input (non-empty, ≥ 8 chars), hashes, and inserts via `store.create_user` (raises `ValueError` on duplicate/invalid).
- `bootstrap_admin()` seeds the env-configured admin on startup **only when auth is on and the users table is empty**.

## Notes / gotchas
- **Opt-in by design:** default-off keeps the disposable single-user mode and the 49-test suite working with no config; auth tests set `PROOFCHECK_AUTH=on` themselves.
- **Stateless sessions:** no session table — the signed cookie *is* the session. Set `PROOFCHECK_SECRET` to keep sessions valid across restarts / multiple workers.
- **Constant-time everywhere it matters:** `compare_digest` for both password and signature checks; dummy hash on unknown users.
- **Separation of concerns:** auth never touches the matching engine; it only wraps the web routes and tags history with a username.
